# 策略筆記：Agent 自舉與工具自生 (Self-Bootstrapping & Tool Augmentation)

這份筆記紀錄了關於如何透過提示詞引導 LLM 自行創生腳本以克服能力瓶頸的深度討論。

## 🎯 核心哲學：從「工人」進化為「工頭」
當 LLM 面對複雜任務、長文本或推理瓶頸時，不應直接處理原始數據，而應優先考慮 **「為自己打造工具」**。這能節省上下文空間並大幅提升執行準確性。

## 🛠️ 三大核心策略 (The Three Pillars)

### 1. 數據過濾 (Data Filtering - "分而治之")
- **場景**：處理大型 RSS XML、日誌檔或超過 1000 字元的原始數據。
- **作法**：引導 Agent 先寫一個 Python/AWK 腳本提取關鍵欄位（如標題、錯誤代碼），再讀取過濾後的結果。
- **目標**：將 500KB 的雜訊轉化為 2KB 的精華，防止上下文爆炸。

### 2. 專用計算 (Specialized Tools - "能力補強")
- **場景**：數學運算、精確排序、跨檔案內容比對、邏輯判斷。
- **作法**：引導 Agent 將邏輯封裝進 Python 腳本執行，並直接獲取計算結果而非自行推導。
- **目標**：消除 LLM 在精確邏輯與數學上的不穩定性。

### 3. 診斷與修復 (Diagnostic & Self-Healing - "環境自覺")
- **場景**：指令連續失敗、權限不足、連線超時。
- **作法**：引導 Agent 寫一個偵測腳本（如 `check_env.sh`）診斷系統狀態，並根據診斷報告修正下一步行動。
- **目標**：從「盲目重試」轉向「邏輯修正」。

## 🚀 未來進化機制：技能升遷 (Skill Promotion)
這是一個讓 Agent 實現自我演進的閉環邏輯：
1. **創生 (Create)**：在 `scripts/` 下產生臨時腳本解決當前問題。
2. **驗證 (Verify)**：腳本執行成功且具備重複使用價值。
3. **升遷 (Promote)**：Agent 建議將腳本移至 `plugins/`，並在 `system.md` 中註冊為常駐技能。
4. **內化 (Internalize)**：下次遇到相同問題時，直接呼叫該插件。

## 📝 實施建議 (Implementation Hints)
在未來的 `system.md` 更新中，應加入以下引導語：
> 「你是一個具備自覺性的工程師。如果你覺得任務太重或數據太雜，請優先考慮寫一個臨時腳本（槓桿）來輔助自己。遵循：思考 -> 寫入 scripts/ -> 執行 -> 讀取精簡結果 -> 決策。」
